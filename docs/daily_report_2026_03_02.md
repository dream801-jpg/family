# 🌙 가족 캘린더 앱 - 일일 작업 보고서

현재 시간은 늦었으므로, 오늘 진행한 작업들을 갈무리하고 현재 상태 및 향후 과제를 정리해 드립니다.

## 🏆 오늘 완성된 주요 기능

1. **설정 화면 UI 및 기능 복구**
   - 역할 선택 (남편/아내), 알림 방식 (소리+진동/진동만/화면만), 알림 필터 (내 일정/가족 일정/상대 휴무 등)가 정상적으로 저장되고 로드되도록 수정했습니다.
   - 모바일 화면에서 설정 모달이 짤리던 현상을 스크롤 기능(`max-height`, `overflow-y`)을 적용해 수정했습니다.

2. **기본 알람(Ringtone) 선택 기능 추가 (진행 중 -> UI 완성)**
   - 앱 내부 하드코딩된 소리가 아닌, **스마트폰 기기 자체에 내장된 기본 알림음(Ringtone)**을 부를 수 있는 **네이티브 안드로이드 플러그인(RingtonePlugin.java)**을 직접 제작하여 앱에 심었습니다.
   - 브라우저에서는 동작하지 않지만, 안드로이드 앱에서 설정의 **[🎵 소리 선택]** 버튼을 누르면 안드로이드 기본 알림음 선택창이 뜨고, 사용자가 고른 소리의 고유 주소(URI)와 이름이 앱에 저장됩니다.

3. **역할(Role) 기반의 맞춤형 푸시 알림 서버로직 고도화**
   - Cloud Functions 서버가 기기마다 저장된 설정(`filterMine`, `filterFamily`, `filterPartner`)을 확인하여 **정말 알림을 받아야 하는 기기에만** 선별적으로 푸시를 보내도록 완벽히 개선했습니다.

## 🚧 다음에 이어서 해야 할 작업 (Next Steps)

현재 "기기 기본 알람 소리를 고르고 저장하는 기능"까지는 완료되었으나, 백그라운드 푸시 알림이 올 때 **실제로 그 소리가 울리게 하는 부분**은 약간의 추가 작업이 필요합니다.

*   **이유:** 안드로이드 운영체제 정책상, 백그라운드 푸시가 사용자가 고른 `content://...` 형태의 기기 알림음으로 울리려면, 앱이 켜질 때마다 해당 고유 소리별로 **새로운 알림 채널(Notification Channel)**을 동적으로 생성해 주어야 합니다.
*   **다음 작업 목표:** 앱 시작 시 Capacitor 플러그인을 사용하여 사용자가 선택한 알림음 URI로 커스텀 알림 채널(`PushNotifications.createChannel`)을 생성하고, 서버(Cloud Functions)에서 푸시를 보낼 때 해당 채널 ID를 타겟팅하도록 연결하는 작업을 가장 먼저 진행하겠습니다.
